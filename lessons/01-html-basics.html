Lesson 01: HTML Basics

Learning Outcomes

By the end of this lesson, you will be able to understand what HTML is and why it exists, identify the basic structure of an HTML document, recognize common HTML tags and understand their purpose, and create your first simple HTML page. These skills form the foundation for all web development work.

Prerequisites

No prior coding experience is required for this lesson. You only need a text editor (like Notepad, TextEdit, or VS Code) and a web browser (Chrome, Firefox, Safari, or Edge) installed on your computer.

What is HTML and Why Does It Matter?

HTML stands for HyperText Markup Language. This name tells us something important about what HTML does. The word "markup" indicates that HTML doesn't create the content itself, but rather marks up or labels content to tell web browsers how to display and organize that content. HTML provides structure to web pages in the same way that a blueprint provides structure to a building.

When you visit a website, your web browser receives HTML code and interprets it to display the page you see. Without HTML, browsers would have no way to understand which parts of your content are headings, which are paragraphs, which are links, and so on. HTML acts as the universal language that allows any browser on any device to display web content in a consistent, structured way.

Understanding HTML matters because it forms the foundation of every website you see on the internet. Whether you're viewing a simple blog or a complex web application, HTML is always present, working behind the scenes to organize and structure the content. Learning HTML gives you the ability to create and modify web pages, which is an essential skill for web development, content creation, and digital communication.

Understanding HTML and CSS as Files

Before diving into writing HTML code, it's important to understand that HTML and CSS are simply text files saved on your computer. These files contain instructions written in special languages that web browsers can read and interpret. Just like you might create a document in a word processor and save it as a .docx file, you create web pages by writing HTML code and saving it as an .html file.

When you create an HTML file, you're creating a plain text file with special formatting. The file extension .html tells your computer and web browser that this file contains HTML code. Similarly, CSS files are saved with the .css extension. These extensions are important because they help your computer identify what type of file it is and which program should open it. When you double-click an HTML file, your computer knows to open it in a web browser rather than a text editor.

The process of working with HTML and CSS files follows a simple workflow. First, you open a text editor and create a new file. You write your HTML or CSS code in this file, then save it with the appropriate extension. Once saved, you can open the file in a web browser to see how it looks. When you want to make changes, you return to the text editor, modify the code, save the file again, and then refresh the browser to see your changes. This cycle of editing, saving, and refreshing is fundamental to web development.

Understanding this file-based workflow matters because it shows that web development doesn't require special software or complex setup. You can create HTML files using any text editor, from simple programs like Notepad on Windows or TextEdit on Mac to more advanced code editors like Visual Studio Code. The code you write is just text, saved in a file that browsers know how to interpret.

Example: Creating and Saving Your First HTML File

This example demonstrates the complete process of creating an HTML file, from opening a text editor to viewing it in a browser. Following these steps helps you understand how HTML files work on your computer.

Start by opening a text editor on your computer. If you're using Windows, you can use Notepad. If you're using a Mac, you can use TextEdit, but make sure to switch it to plain text mode by going to Format and selecting "Make Plain Text". Alternatively, if you have Visual Studio Code or another code editor installed, you can use that instead.

Create a new file in your text editor. Type or copy the following simple HTML code into the file:

<!DOCTYPE html> <html lang="en"> <head> <title>My First File</title> </head> <body> <h1>Hello from my HTML file!</h1> </body> </html>

Now save this file. Choose File and then Save from your editor's menu. When the save dialog appears, navigate to a folder where you want to keep your web development projects. Create a new folder called "my-website" if you don't have one yet. For the filename, type "index.html" - make sure to include the .html extension. This extension is crucial because it tells your computer this is an HTML file.

After saving, navigate to the folder where you saved your file using your computer's file explorer or finder. You should see a file named "index.html". Notice how the file might have a browser icon, indicating your computer recognizes it as a web page. Double-click this file, and it should open in your default web browser, displaying "Hello from my HTML file!" as a heading.

Now let's practice the save and refresh cycle. Return to your text editor where you have the HTML file open. Change the heading text from "Hello from my HTML file!" to "I changed this text!". Save the file again using File and then Save, or by pressing Ctrl+S on Windows or Cmd+S on Mac. This updates the file on your computer with your new changes.

Return to your web browser where the HTML file is displayed. Click the refresh button or press F5 (or Cmd+R on Mac) to reload the page. You should now see "I changed this text!" instead of the original heading. This demonstrates how changes to your HTML file only appear in the browser after you save the file and refresh the page. Remembering to save and refresh is essential - your browser doesn't automatically detect file changes, so you need to refresh to see updates.

The Basic Structure of an HTML Document

Every HTML document follows a specific structure that tells the browser what type of document it is and how to interpret the content. This structure consists of several key elements that work together to create a valid web page.

The first line of every HTML document should be a document type declaration, written as <!DOCTYPE html>. This tells the browser that it's reading an HTML5 document, which is the current standard. While browsers are often forgiving if this line is missing, including it ensures your page will display correctly across all browsers and devices.

After the document type declaration, every HTML document is wrapped in an <html> tag. This tag tells the browser where the HTML content begins and ends. Inside the <html> tag, there are two main sections: the <head> section and the <body> section. The head section contains information about the document that isn't displayed on the page itself, such as the page title that appears in the browser tab, links to stylesheets, and other metadata. The body section contains all the visible content that users will see when they visit your page.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My First Page</title> </head> <body> <h1>Welcome to My Website</h1> <p>This is my first HTML page.</p> </body> </html>

The lang="en" attribute in the opening <html> tag specifies that the document is written in English. This helps screen readers and search engines understand the language of your content. The <meta charset="UTF-8"> line in the head section tells the browser which character encoding to use, ensuring that special characters and international characters display correctly.

Understanding HTML Tags and Elements

HTML uses tags to mark up content. Tags are written using angle brackets, with most tags coming in pairs: an opening tag and a closing tag. The opening tag begins with a less-than symbol followed by the tag name, and ends with a greater-than symbol. The closing tag is identical but includes a forward slash before the tag name. The content that appears between the opening and closing tags is what the browser will display or apply that tag's meaning to.

For example, if you want to create a paragraph, you would use the <p> tag. To mark where your paragraph begins, you write <p>, then type your paragraph content, and finally close the paragraph with </p>. The browser interprets this and displays your text as a paragraph with appropriate spacing. This pairing of opening and closing tags creates what we call an HTML element.

Some tags are self-closing, meaning they don't require a closing tag. These are typically used for elements that don't contain content, such as line breaks or images. For example, <br> creates a line break and doesn't need a closing tag. However, in HTML5, you can also write it as <br /> if you prefer the explicit self-closing syntax.

Understanding tags is fundamental because every piece of content on a web page is marked up with tags. These tags don't just affect how content looks visually, but more importantly, they provide semantic meaning. When you use an <h1> tag, you're not just making text large and bold - you're declaring that this text is the main heading of the page. This semantic meaning is important for accessibility, search engines, and other tools that read web pages.

Common HTML Tags You'll Use Regularly

While HTML has many tags available, certain tags appear in almost every web page you'll create. Understanding these fundamental tags gives you the ability to create meaningful, well-structured web pages.

Heading tags, written as <h1> through <h6>, are used to create headings of different importance levels. The <h1> tag represents the most important heading on the page and is typically used for the main title. Each subsequent number represents a heading of lesser importance, allowing you to create a hierarchy of information. You should use only one <h1> per page, as this represents the main topic or title of the entire page. Using headings in a logical order helps users and search engines understand the structure of your content.

The paragraph tag, written as <p>, is used to wrap blocks of text content. When the browser encounters a paragraph tag, it displays the text with appropriate spacing before and after, creating visual separation between different paragraphs. This makes your content more readable and organized. Every paragraph you write should be wrapped in opening and closing paragraph tags.

The anchor tag, written as <a>, creates hyperlinks that allow users to navigate to other pages or sections. The anchor tag requires an attribute called href that specifies the destination of the link. Attributes provide additional information about HTML elements and are written inside the opening tag. For a link, you write the destination URL in the href attribute, like this: <a href="https://example.com">Click here</a>. The text between the opening and closing anchor tags is what users will see and click on.

Other essential tags you'll encounter frequently include the image tag <img> for displaying pictures, the strong tag <strong> for emphasizing important text, and the emphasis tag <em> for italicized emphasis. Each of these tags serves a specific purpose in marking up your content meaningfully.

Example: Using Common HTML Tags

This example demonstrates how to use several common HTML tags together to create a simple but complete web page. Seeing these tags work together helps you understand how they combine to create structured content.

Create a new HTML file and add the basic document structure we learned earlier. Inside the body section, let's add content using different tags. Start with an <h1> tag for the main title of your page. Following best practices, use only one <h1> per page.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My Learning Journey</title> </head> <body> <h1>Learning Web Development</h1> <h2>Getting Started</h2> <p>I'm learning HTML and CSS to build my own websites. This is exciting!</p> <h2>What I've Learned</h2> <p>So far, I understand how HTML tags work. The <strong>strong</strong> tag makes text bold, and the <em>emphasis</em> tag makes text italic.</p> <p>Visit <a href="https://www.example.com">this example website</a> to see more.</p> </body> </html>

Save this file with an .html extension and open it in your browser. You'll see the heading "Learning Web Development" displayed prominently, followed by two smaller headings. Notice how the paragraphs provide content under each heading, and how the link appears in a different color, indicating it's clickable. The <strong> text appears bold, and the <em> text appears in italics.

Experiment with this example by changing the text content, adding more paragraphs, or including additional headings. Each time you make a change, save the file and refresh your browser to see the results. This practice helps you understand how each tag affects the display of your content.

Worked Example: Creating Your First HTML Page

Creating your first HTML page involves several steps, and understanding why each step matters will help you remember the process. We'll create a simple page that displays a greeting and a short paragraph.

First, open a text editor on your computer. You can use any plain text editor - Notepad on Windows, TextEdit on Mac (making sure it's in plain text mode), or a code editor like Visual Studio Code if you have one installed. Create a new file and save it with the name "my-first-page.html". The .html extension is important because it tells your computer and browser that this file contains HTML code.

Now we'll write the HTML structure. Start with the document type declaration on the first line. This tells the browser we're using HTML5. On the next line, add the opening <html> tag with the language attribute. Then create the head section, which will contain metadata about our page. Inside the head, add a meta tag for character encoding, and a title tag with text that describes your page. The title text will appear in the browser tab when someone visits your page.

After closing the head section with </head>, open the body section where your visible content will go. Inside the body, add an <h1> heading tag with a greeting message. Follow this with a paragraph tag containing a sentence about what you're learning. Finally, close all the tags in the proper order: close the body, then close the html tag.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My First HTML Page</title> </head> <body> <h1>Hello, World!</h1> <p>I'm learning HTML and this is my first web page.</p> </body> </html>

To view your page, navigate to the folder where you saved your HTML file using your computer's file explorer. Double-click the file, and it should open in your default web browser. You should see your heading and paragraph displayed on the page. If something doesn't appear correctly, check that all your tags are properly closed and that you saved the file with the .html extension.

This simple example demonstrates the fundamental structure that every HTML document follows. As you continue learning, you'll add more content and tags, but the basic structure of document type, html tag, head section, and body section will remain constant across all your HTML pages.

Practice Opportunity

Create an HTML page about a topic that interests you. Include a title in the head section, one main heading using the h1 tag, and at least three paragraphs using paragraph tags. Add a second-level heading using h2 somewhere in your content. Include one link to an external website using the anchor tag. Save your file and open it in a web browser to see how it looks.

When you're finished, compare your code structure to the worked example above. Make sure all your tags are properly opened and closed, and that your content appears in the browser as you expect. If something doesn't display correctly, check that you've closed all your tags and that your HTML structure follows the basic format we discussed.

Summary of Key Concepts

HTML provides the structural foundation for all web pages. It uses tags to mark up content, giving browsers instructions on how to organize and display information. Every HTML document follows a consistent structure with a document type declaration, an html element containing head and body sections, and proper use of tags to mark up content meaningfully.

The tags you've learned in this lesson - headings, paragraphs, links, and the basic document structure - form the building blocks you'll use in every web page you create. Understanding why HTML exists and how these basic tags work prepares you to build more complex pages as you continue learning. Remember that HTML is about meaning and structure, not visual appearance. The visual styling comes later when we learn CSS.

Next Steps

Once you're comfortable creating basic HTML pages with headings, paragraphs, and links, you're ready to move on to Lesson 02, which covers semantic HTML. Semantic HTML involves choosing the most meaningful tags for your content, which creates better structure and improves accessibility for all users.
